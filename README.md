# Meeting Room Booking Service

Сервис бронирования переговорных комнат, разработанный на фреймворке FastAPI с использованием PostgreSQL в качестве базы данных и SQLAlchemy (асинхронный режим) в качестве ORM.

## Архитектура проекта

Проект спроектирован с разделением зон ответственности по слоям (Layered Architecture):
- **Routers**: Обработка HTTP-запросов, валидация входных данных через Pydantic-схемы.
- **Service**: Реализация бизнес-логики приложения.
- **Data Access Object (DAO)**: Абстракция над базой данных, выполнение CRUD-операций.
- **Models**: Декларативные модели SQLAlchemy.
- **Schemas**: Pydantic-модели для валидации данных и сериализации ответов.

## Варианты запуска проекта

Конфигурация приложения управляется переменными окружения через файл `.env`. Способ подключения к базе данных зависит от выбранного режима работы.

### Вариант 1: Полный запуск в Docker (Рекомендуемый)

В данном режиме все компоненты (FastAPI-приложение и СУБД PostgreSQL) запускаются внутри единой изолированной сети Docker.

1. Переименуйте файл `.env.template` в `.env` и установите в нём следующие параметры:

```

```text

```env
db_login=your_db_login
db_password=your_db_password
db_name=your_db_name
JWT_SECRET_KEY=jwt_secret_key
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=60
db_port=5432
db_host=postgres


```

*Здесь `postgres` — это имя сервиса базы данных в файле `docker-compose.yml`, которое Docker разрешает в IP-адрес контейнера.*

2. Выполните команду для сборки образов и запуска контейнеров:
```bash
docker-compose up --build

```


3. В процессе запуска приложения, контейнер автоматически:
* Дождется доступности порта базы данных с помощью утилиты `netcat`.
* Запустит миграции Alembic.
* Выполнит заполнение базы данных тестовыми комнатами и пользователями.
* Запустит сервер Uvicorn на порту `8989`.


---

### Вариант 2: Локальный запуск приложения (База данных в Docker)

Этот режим используется для локальной разработки и отладки кода приложения, когда FastAPI запускается непосредственно на хост-машине (например, через Poetry или IDE), а база данных изолирована в контейнере.

Для изоляции базы данных используется отдельный конфигурационный файл `docker-compose_postgres.yaml`.

#### Нюансы конфигурации сети

Когда приложение работает локально на хосте, оно не имеет доступа к внутренней сети Docker по имени контейнера `postgres`. Для подключения необходимо обращаться к `localhost` (или `127.0.0.1`), так как порт СУБД проброшен на хост-машину.


1. Настройте файл `.env` для локального подключения (В случае недоступности порта 5432, выберите свободный порт):
```env
DB_HOST=localhost
DB_PORT=5432

```

2. Запустите только контейнер базы данных:
```bash
docker-compose -f docker-compose_postgres.yaml up -d --build

```


3. Установите зависимости, запустите миграции и заполнение базы данных вручную из корневой директории проекта:
```bash
poetry install
poetry run alembic upgrade head
python init_db.py

```


4. Запустите локальный сервер разработки:
```bash
poetry run uvicorn src.main:app --host 127.0.0.1 --port 8989 --reload

```

## Примеры работы с API

После успешного запуска проекта интерактивная документация Swagger UI доступна по адресу: `http://127.0.0.1:8989/docs`.

Ниже приведены примеры основных HTTP-запросов для работы с сервисом.

### 1. Регистрация нового пользователя

* **Запрос:** `POST /auth/register`
* **Тело запроса (JSON):**
```json
{
  "username": "user123",
  "password": "Password123*",
  "name": "Pavel"
}

```


* **Успешный ответ (`201 Created`):**
```json
{
  "message": "Пользователь user123 успешно зарегистрирован"
}

```


### 2. Аутентификация и получение токена

* **Запрос:** `POST /auth/login`
* **Тело запроса (Form Data):**
```ini
username=user123
password=Password123

```


* **Успешный ответ (`200 OK`):**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}

```



### 3. Получение списка комнат

* **Запрос:** `GET /bookings/rooms`
* **Заголовки:** `Authorization: Bearer <access_token>`
* **Успешный ответ (`200 OK`):**
```json
[
  {
    "id": 1,
    "name": "Переговорная 1",
    "description": "Зал для презентаций и видеоконференций — с видеокамерой, микрофонами, экраном и Wi-Fi"
  },
  {
    "id": 2,
    "name": "Переговорная 2",
    "description": "Стандарт на 6–8 человек"
  }
]

```


### 4. Получение списка доступных комнат на указанную актуальную дату 

* **Запрос:** `GET /bookings/room-available/{booking_date}`
* **Заголовки:** `Authorization: Bearer <access_token>`
* **Успешный ответ (`200 OK`):**
```json
[
  {
    "id": 1,
    "name": "Переговорная 1",
    "description": "Зал для презентаций и видеоконференций — с видеокамерой, микрофонами, экраном и Wi-F",
    "free_slots": [
      "09:00-11:00",
      "11:00-13:00",
      "13:00-15:00",
      "15:00-17:00",
      "17:00-19:00"
    ]
  },
  {
    "id": 2,
    "name": "Переговорная 2",
    "description": "Стандарт на 6–8 человек",
    "free_slots": [
      "09:00-11:00",
      "11:00-13:00",
      "13:00-15:00",
      "15:00-17:00",
      "17:00-19:00"
    ]
  }
]

```


### 5. Создание бронирования переговорной комнаты. 
Интервал выбирается из списка ALLOWED_SLOTS из `settings.py`

```python
ALLOWED_SLOTS = [
    "09:00-11:00",
    "11:00-13:00",
    "13:00-15:00",
    "15:00-17:00",
    "17:00-19:00",
]

```

* **Запрос:** `POST /bookings`
* **Заголовки:** `Authorization: Bearer <access_token>`
* **Тело запроса (JSON):**
```json
{
  "room_id": 1,
  "booking_date": "2026-06-17",
  "time_slot": "09:00-11:00"
}

```

* **Успешный ответ (`201 Created`):**
```json
{
  "message": "Комната 1 забронирована на дату 2026-06-17 на интервал 09:00-11:00"
}

```


### 6. Отмена бронирования.
Сотрудник может отменять только свои, админ - любые

* **Запрос:** `DELETE /bookings/{booking_id}`
* **Заголовки:** `Authorization: Bearer <access_token>`

* **Успешный ответ (`200`):**
```json
{
  "status": "success",
  "message": "Бронирование успешно отменено"
}

```


## Тестирование

Проект покрыт автоматическими тестами. Для изоляции тестовой среды при запуске тестов конфигурация автоматически использует тестовую базу данных.

Запуск тестового набора выполняется командой:

```bash
poetry run pytest

```
